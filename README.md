# 알고먹자

식단과 운동을 기록하고 일일 칼로리·영양소, 운동 기록, 체중 목표를 관리할 수 있는 Java Swing 기반 데스크톱 프로그램입니다.

## 프로젝트 소개

알고먹자는 음식 섭취와 운동 기록을 한곳에서 관리하고, 일일 영양소와 목표 진행 상황을 확인할 수 있도록 만든 건강 관리 프로그램입니다. 일반 사용자는 식단·운동·신체정보·목표를 기록할 수 있고, 관리자는 공지사항과 첨부파일을 관리할 수 있습니다.

2025년 4인 팀 프로젝트로 개발했으며, 프로젝트 종료 후에는 실행 환경, 데이터 처리 안정성, 인증 보안, 기능 오류와 UI/UX를 개인적으로 개선했습니다.

## 주요 기능

### 식단·영양 관리

- 아침·점심·저녁·간식 유형별 음식 검색과 중량 조절
- 음식별 칼로리, 탄수화물, 단백질, 지방 조회
- 식단 헤더와 상세 음식의 단일 트랜잭션 저장
- 일일 섭취 열량과 영양소 합계 표시

### 운동·캘린더

- 운동 카테고리·이름 검색과 상세 조회
- 운동 시간과 현재 체중을 이용한 예상 소모 칼로리 계산
- 운동 기록 저장
- 월간 캘린더 이동, 날짜 선택, 선택일 식단·운동 기록 조회

### 신체정보·목표

- 키, 몸무게, 체지방률, 체지방량, 골격근량 기록
- 시작 체중, 목표 체중, 목표 기간 설정
- 체중 변화 방향과 목표 진행률 표시

### 회원·공지

- 회원가입, 로그인, 회원정보 수정
- 휴대폰 가입정보 대조를 통한 아이디 찾기와 비밀번호 재설정
- PBKDF2 기반 비밀번호 저장·검증
- 사용자 공지 조회와 관리자 공지 작성·수정·삭제
- 공지 첨부파일 복사, 다운로드, 삭제 정리

## 담당 역할

- 팀장으로서 기능 범위와 개발 일정을 조율했습니다.
- 음식 검색, 상세 중량 조절, 담은 음식 관리, 식단 저장과 영양소 집계 흐름을 담당했습니다.
- 신체정보를 활용한 체중 목표 설정·조회 화면을 담당했습니다.
- 회원가입, 로그인, 회원정보 관리 기능을 담당했습니다.
- 운동 관리와 캘린더 기록 조회는 팀원과 공동으로 개발했습니다.
- 데이터베이스는 팀원들과 공동 설계했으며 초기 테이블 구조와 데이터 흐름의 방향을 조율했습니다.

## 프로젝트 종료 후 개인 개선

- Eclipse 프로젝트를 JDK 21 환경에서 다시 빌드할 수 있도록 외부 라이브러리와 리소스 경로를 정리했습니다.
- DB 접속 정보를 로컬 설정 파일로 분리하고 공개 스키마에서 관리자 자격 증명 seed를 제거했습니다.
- 회원 비밀번호를 PBKDF2 형식으로 저장·검증하고 계정 찾기를 비밀번호 재설정 방식으로 변경했습니다.
- 식단 저장을 하나의 JDBC 트랜잭션으로 묶어 헤더와 상세 데이터의 부분 저장 가능성을 줄였습니다.
- 월간 캘린더와 선택 날짜의 식단·운동 기록 조회를 보완했습니다.
- 공지사항의 다건 목록, 긴 본문, 첨부파일 저장·정리 흐름을 개선했습니다.
- 고정 440×956 화면 안에서 주요 패널의 카드, 스크롤, 빈 상태와 입력 화면 배치를 정리했습니다.
- 현재 DAO가 사용하는 테이블·컬럼을 기준으로 신규 환경용 `schema.sql`을 정리했습니다.

## 실행 환경 및 실행 방법

### 필요 환경

| 항목 | 기준 |
|---|---|
| JDK | Eclipse Temurin JDK 21 |
| IDE | Eclipse IDE for Java Developers |
| DB | MySQL 8.0 |
| JDBC Driver | MySQL Connector/J |
| 프로젝트 형식 | Eclipse Java Project |

### 실행 순서

1. 저장소를 내려받고 Eclipse에서 `Existing Projects into Workspace`로 가져옵니다.
2. MySQL Connector/J를 내려받아 `lib/mysql-connector-j.jar` 이름으로 배치합니다.
3. MySQL에서 [`sql/schema.sql`](sql/schema.sql)을 실행해 신규 개발용 스키마를 구성합니다.
4. [`src/db.properties.example`](src/db.properties.example)을 복사해 로컬 `src/db.properties`를 만들고 접속 정보를 입력합니다.
5. Eclipse에서 JDK가 Java 21로 설정되었는지 확인합니다.
6. `src/main/MainFrame.java`를 Java Application으로 실행합니다.

관리자 계정은 공개 스키마에 포함되지 않습니다. 관리자 기능을 확인하려면 로컬 환경에서 PBKDF2 인코딩 비밀번호를 사용하는 계정을 별도로 준비해야 합니다.

현재 스키마와 DAO의 테이블·컬럼 매핑은 [데이터베이스 스키마와 DAO 매핑](docs/database-schema.md)에 정리했습니다.
