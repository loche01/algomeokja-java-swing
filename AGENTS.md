# AGENTS.md

이 프로젝트는 기존 Eclipse 기반 Java Swing 식단 관리 프로그램이며, macOS에서 복구하여 포트폴리오용으로 정리 중이다.

## 프로젝트 환경

* Java Swing 기반 기존 프로젝트 구조 유지
* Eclipse Java Project 구조 유지
* Eclipse Temurin JDK 21 사용
* MySQL 스키마 `almja` 사용
* 현재 작업 브랜치는 `codex-audit`

## 작업 원칙

* 한 번의 요청에는 하나의 명확한 목표만 처리
* 먼저 원인을 분석하고, 사용자 승인 후 수정
* 관련 없는 파일 수정 금지
* 요청하지 않은 대규모 리팩터링 금지
* Maven, Gradle, Spring, Spring Boot로 임의 변환 금지
* 기존 UI, 패널 이름, 화면 전환 구조 최대한 보존
* 실행하지 못한 기능을 정상이라고 보고하지 않기
* 정적 분석과 실제 실행 검증 결과를 구분하기

## Git 규칙

* `main`, `recovery` 브랜치를 직접 수정하지 않기
* 별도 작업 브랜치에서만 수정
* 수정 전후 `git status`와 `git diff` 확인
* 의도하지 않은 줄바꿈 전체 변경 금지
* 문제 하나를 해결하고 검증한 뒤 커밋
* Git 커밋 메시지는 한국어로 작성
* 사용자 승인 없이 commit 또는 push 금지

## 보안 규칙

* `src/db.properties`를 읽어서 출력하거나 변경하지 않기
* DB 비밀번호, GitHub 토큰, 개인정보 출력 금지
* `src/db.properties`를 Git 추적 대상으로 추가하지 않기
* API 키나 토큰을 코드, 문서, 로그에 기록하지 않기

# 알고먹자 장기 안정화 작업 규칙

- 장기 작업 브랜치는 `codex-stabilization`이다.
- 작업은 계획 순서대로 하나씩 수행한다.
- 서로 다른 기능은 별도 한국어 커밋으로 나눈다.
- 코드 수정 전 관련 파일과 호출 흐름을 분석한다.
- 관련 없는 파일을 수정하지 않는다.
- 자동 포맷팅으로 파일 전체 줄바꿈을 변경하지 않는다.
- 각 작업 후 `git diff --check`와 diff를 확인한다.
- 작업 결과를 `docs/CODEX_WORKLOG.md`에 기록한다.
- push와 `codex-audit`/`main` 병합은 하지 않는다.
- `src/db.properties`를 읽거나 출력하지 않는다.
- 개인정보를 콘솔에 출력하지 않는다.
- DB 스키마와 SQL 파일을 임의로 변경하거나 실행하지 않는다.
- `MainFrame`의 440x956 고정 크기를 유지한다.
- 새로운 외부 라이브러리를 임의로 추가하지 않는다.
- 기존 비밀번호를 조회하거나 사용자에게 표시하지 않는다.
- 비밀번호 찾기는 본인 확인 후 새 비밀번호 설정 방식으로 구현한다.
- 휴대폰 번호는 계정 찾기용으로 유지한다.
- 생년월일과 성별은 이번 작업에서 삭제하지 않는다.
- 키보드 입력 언어를 강제로 바꾸지 않는다.
- 캘린더는 월간 달력, 월 이동, 날짜 선택, 선택일 식단·운동 기록 표시를 목표로 한다.

## 자동 진행 중단 조건

- DB 테이블이나 컬럼 변경이 필요한 경우
- 기존 데이터 삭제가 필요한 경우
- 사용자 인증 정책을 코드만으로 확정할 수 없는 경우
- 현재 작업과 무관한 컴파일 오류가 발생한 경우
- 계획보다 수정 범위가 크게 확장되는 경우
- Eclipse 화면 또는 실제 DB 데이터 확인이 필요한 경우
- 체크포인트 1, 2, 3에 도달한 경우
