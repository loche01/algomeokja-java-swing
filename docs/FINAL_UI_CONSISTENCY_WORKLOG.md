## 2026-07-13 / 체크포인트 1 로그인·계정 찾기·비밀번호 UI 보완

- 로그인 초기화: `MainFrame`이 재사용하는 `LoginPanel`을 필드로 보관한다. 로그인 실패는 아이디를 유지하고 비밀번호와 보기 상태만 초기화하며, 성공은 저장 아이디 정책 처리 후 비밀번호와 보기 상태를 초기화한다.
- 로그아웃 초기화: 기존 세션 로그아웃 뒤 `showLoginAfterLogout()`을 사용한다. 아이디 저장이 선택된 경우 저장된 아이디를 다시 표시하고, 선택되지 않은 경우 아이디를 비운다. 비밀번호는 항상 비우고 숨김 상태로 복구한다.
- 이메일 찾기 제거: 동작과 연결이 없던 `FindIdPwEmail`을 삭제하고 휴대폰 찾기 화면의 이메일 탭과 단일 탭 형태를 제거했다. 가입 이메일, 형식·중복 검사와 회원정보 이메일은 유지했다.
- 입력 필터: `PasswordDocumentFilter`가 기존 정책의 ASCII 문자만 허용하고 최대 20자로 제한한다. 잘못된 직접 입력과 붙여넣기는 전체 거부하고 beep로 알리며 입력값이나 거부 문자를 출력하지 않는다.
- 필터 적용: 회원가입 비밀번호·확인, 휴대폰 재설정의 새 비밀번호·확인, 회원정보의 새 비밀번호·확인에만 적용했다. 로그인 비밀번호와 회원정보 현재 비밀번호에는 적용하지 않았다.
- 보기·숨기기: `PasswordVisibilityToggle`을 추가해 로그인, 회원가입 2개, 재설정 2개, 회원정보 3개 필드가 독립적으로 동작한다. 각 필드의 기본 echo 문자를 보존하고 화면 재진입·초기화 시 숨김으로 복구한다.
- 회원정보 UI: 내 정보의 빨간 텍스트형 `수정` 버튼을 테두리가 보이는 `회원정보 수정` 보조 버튼으로 바꾸고 동작 없는 중복 리스너를 제거했다. 수정 화면은 `변경사항 저장` 버튼을 폼 폭에 맞추고 세 비밀번호 필드 옆에 52px 보기 버튼을 배치했다.
- 인증 유지: 일반 정보만 수정할 때 현재 비밀번호가 필요 없는 정책, 새 비밀번호 입력 시 현재 비밀번호 필수, 현재 비밀번호 오류 시 전체 저장 차단과 기존 PBKDF2 저장 흐름을 변경하지 않았다.

## 2026-07-13 / 체크포인트 1-B 인증 및 회원정보 UI 마감

- 공통 스타일: `AppTheme`에 연한 초록 배경, 흰 카드, 초록 계열 주요색, 텍스트·경계선·오류 색상, Java 논리 폰트, 주요·보조·로그아웃 버튼과 입력 필드 스타일을 정의했다. 적용 범위는 이번 대상 화면으로 제한했다.
- 내 정보: 회원정보·신체정보 섹션 제목과 정보 행을 같은 기준으로 정렬하고, `회원정보 수정`과 `신체정보 수정` 버튼에 동일한 보조 스타일을 적용했다. 빨간색 강조, 비밀번호 `*****` 행, 동작 없는 컴포넌트와 리스너를 제거했다.
- 회원정보 수정: 화면 안에서 끊겨 보이던 `RoundedComponent` 카드 래퍼를 이 화면에서만 일반 흰 카드로 교체했다. `회원정보 수정` 제목과 `내 정보로` 버튼을 같은 헤더에 배치하고, 일반 입력 4개와 비밀번호 입력 3개의 높이·폭·간격을 통일했다.
- 저장 영역: 비밀번호 보기 버튼은 각 입력과 겹치지 않는 52px 폭을 유지하고, 규칙 안내가 두 줄 안에서 보이도록 했다. `변경사항 저장` 버튼은 비밀번호 입력과 보기 버튼을 합친 영역에 맞췄으며 카드 하단 여백을 확보했다.
- 계정 찾기: 상단 제목·설명을 별도 카드로 정리하고 아이디 찾기와 비밀번호 재설정 영역을 같은 카드·입력·버튼 스타일로 통일했다. 찾은 아이디는 입력 필드와 구분된 결과 카드에서만 표시하며 콘솔이나 문서에는 기록하지 않는다.
- 재진입 초기화: 로그인 화면의 계정 찾기 진입은 `MainFrame.showFindIdPwPanel()`을 거친다. 이 메서드가 `resetForEntry()`를 먼저 호출해 이름 2개, 휴대폰 번호 2개, 재설정용 ID, 두 사용자 확인 상태, 찾은 아이디 결과와 포커스를 초기화한다.
- 기능 유지: 계정 찾기 DAO 호출, 회원정보 일반 수정·비밀번호 변경 조건, 현재 비밀번호 오류 시 전체 저장 차단, PBKDF2 인증·저장 흐름과 기존 비밀번호 규칙을 변경하지 않았다.
