# 알고먹자 최종 고도화 계획

## 기준

- 작업 브랜치: `codex-final-polish`
- 기준 브랜치: `codex-audit`
- 기준 커밋: `4089009 문서: 통합 안정화 결과 및 검증 내역 반영`
- 화면 기준: `MainFrame` 440x956 고정 크기와 기존 Java Swing/Eclipse 프로젝트 구조 유지

## 전체 목표

기존 안정화 단계에서 사용자 검증을 통과한 기능과 DAO 계약을 유지하면서 화면 가독성, 조작 영역, 플랫폼 독립성, 보안과 데이터 일관성을 순차적으로 고도화한다. 각 단계는 기능 변경과 실행 검증 범위를 분리해 기록하고, 실행하지 못한 항목은 완료로 표현하지 않는다.

## 단계별 범위

1. 식단 UI/UX
   - 식단 홈 카드, 음식 검색·목록, 음식 상세·중량·담기 화면을 440px 폭에 맞게 정리한다.
   - 음식 검색·상세·담기·저장·`HomeMeal` 복귀와 기존 DAO 호출 계약은 유지한다.
2. 공지사항 실행 검증
   - 다건 목록, 긴 본문, 카드 빈 여백과 관리자 공지 UI를 실제 데이터·권한 범위에서 검증한다.
3. 로그인 전 정상 로그와 `finalize()` 경고
   - 로그인 전 정상 상태의 불필요한 로그와 `DBConnectionMgr.finalize()` 제거 예정 경고를 별도 범위에서 다룬다.
4. 식단 저장 트랜잭션
   - 식단 헤더와 상세 저장의 원자성을 검토한다. DB 계약과 실패 정책 확인 전에는 구현하지 않는다.
5. 비밀번호 단방향 해시
   - 기존 평문 호환, 마이그레이션과 로그인·재설정 정책을 확정한 뒤 별도 보안 단계로 진행한다.

## 체크포인트 1: 식단 UI/UX

### 현재 범위

- `HomeMealPanel`의 아침·점심·저녁·간식 카드 구조와 간격 정리
- 식사 유형, 현재 열량과 음식 추가 조작 영역 구분
- `FoodListPanel`의 선택 식사 유형, 검색, 목록, 담은 목록과 저장·복귀 우선순위 정리
- `FoodInfoPanel`의 선택 음식, 중량, 담기와 목록 복귀 조작 구분
- 세로 스크롤 유지, 가로 스크롤 차단, 빈 목록 안내와 긴 음식명 처리
- macOS에서 보이지 않는 Windows 절대 이미지 경로 의존 제거

### 분석된 연결 구조

- `HomeMealPanel`의 네 식사 버튼은 동일한 `MainUserPanel.foodListPanel`에 `setMealType()`을 호출한 뒤 `showPanel("foodList")`로 이동한다.
- `FoodListPanel`의 음식 행은 `FoodInfoPanel.updateFoodInfo()` 뒤 `showPanel("foodInfo")`를 호출한다.
- `FoodInfoPanel`의 담기는 `FoodListPanel.addToFavoriteList()` 또는 `updateFavoriteItem()`을 호출하고 담은 목록 탭으로 복귀한다.
- `FoodListPanel.saveMealToDatabase()`는 기존 `MealDAO.insertMeal()`과 `MealLogDAO.insertMealLogs()` 순서를 사용하며 성공한 경우에만 안내·초기화 후 `HomeMeal` 열량 갱신과 복귀를 실행한다.

### 현재 UI 문제와 개선 방향

- 식단 홈 네 카드가 중복 코드와 절대 이미지 경로를 사용해 macOS에서 빈 영역이 발생하고 제목·열량·추가 버튼의 관계가 약하다. 공통 카드 구성을 사용해 텍스트 기반 조작 영역으로 통일한다.
- 사용자 이름과 안내 문구가 한 행의 고정 폭에 분리되어 긴 이름에서 겹칠 수 있다. 두 행으로 분리한다.
- 음식 검색 버튼이 Windows 절대 이미지 경로에 의존하고, 선택한 식사 유형이 화면에 표시되지 않는다. 텍스트 검색 버튼과 식사 유형 안내를 사용한다.
- 음식 목록의 세로 스크롤 정책이 `NEVER`이고 가로 스크롤 정책이 명시되지 않았다. 세로 `AS_NEEDED`, 가로 `NEVER`로 정리한다.
- 빈 검색 결과와 빈 담은 목록이 큰 공백으로 표시되고 `+`·`-` 버튼의 의미가 불명확하다. 안내 문구와 `담기`·`빼기` 문구를 사용한다.
- 음식 상세의 음식명 bounds가 카드 폭을 넘고 `<` 버튼과 중량 필드 역할이 불명확하다. 전체 폭 안에서 이름을 줄바꿈하고 `목록으로`, 중량 조절 안내와 `담은 목록에 반영` 문구를 사용한다.
